# Product

## Register

product

## Users

**技术写作者** — 习惯 Markdown，需要本地管理大量技术文档。在多个编辑器间切换，需要统一的渲染体验。

**学生与研究者** — 需要笔记 + 导出 + 纯离线。图书馆、教室、咖啡馆——环境多变，不能依赖网络。

**隐私敏感者** — 数据绝不上云。笔记是思考的延伸，不应交给任何第三方。

**极简主义者** — 反感功能臃肿的笔记工具。要的是"写"和"查"，不是"社交"和"仪表盘"。

**共同上下文**：用户打开 MarkLuck 就像走进一间安静的书房。坐下，开始思考。工具退后，内容浮现。

## Product Purpose

MarkLuck 让用户用纯 `.md` 文件管理笔记——文件夹即笔记本，任何文本编辑器都能打开。数据永不锁死在专有格式中。

**成功标志**：用户忘记 MarkLuck 的存在。他们只是在思考、在写、在查。工具是透明的媒介，不是目的地。

## Brand Personality

**无感 · 真诚 · 精确**

- **无感（Invisible）**：工具消失在任务背后。每一个交互都贴合用户的直觉预期。不打断心流，不需要"学习"。用户感觉不到 UI 的存在，只感觉到自己的思考在流动。
- **真诚（Honest）**：文件就是文件，结构就是结构。不伪装，不粉饰。`**粗体**` 就是粗体，文件夹就是文件夹。所见即所得，所见即所有。
- **精确（Precise）**：每一个像素都有工作。间距遵循数学规律，对齐精确到子像素。无冗余、无装饰、无"差不多"。

## Design Philosophy

**功能即装饰（Function as Ornament）**

拒绝"为了让界面好看而加装饰"，也拒绝"功能 UI 就该千篇一律"。功能分区、结构标识、交互控件本身就是画面的构成元素。文件树不只是导航——它是立面构图中的垂直结构线。格式工具栏不只是按钮组——它是水平方向的几何节奏。区块标记不只是语法提示——它是画面中的标点符号。

这一哲学直接源自工业/建筑设计传统——Dieter Rams 的"少，但更好"、安藤忠雄的清水混凝土结构诚实性、深泽直人的无意识设计。建筑不掩饰梁柱，而是让结构成为美感本身。MarkLuck 不掩饰功能，而是让功能分区成为视觉构成。

## Anti-references

以下设计风格是 MarkLuck 明确拒绝的：

- **SaaS 模板感**：蓝白渐变、圆角卡片网格、Hero 大标题 + CTA 按钮、标准 12 列栅格。千篇一律的"现代 SaaS"外观。
- **企业软件感**：密集数据表格、多层嵌套菜单、灰色底色遍布。把复杂度直接映射为视觉重量。
- **终端极客感**：全黑背景 + 绿色等宽字体 + 纯文本命令行美学。小众且排他。
- **装饰性设计**：为了视觉冲击而添加的渐变、阴影、动画、插画。功能不需要的，就不应该有。

MarkLuck 的视觉语言必须是它自己的——不是任何一个现有类别的子集。

## Design Principles

1. **结构即界面** — 布局由功能分区自然驱动，不套模板。文件树、编辑器、导航树三者构成的空间关系本身就是设计。
2. **静默，不是静音** — 界面退后但不消失。色彩和几何提供微妙的视觉韵律，但不主动吸引注意力。用户工作时感知不到 UI 的存在。
3. **数学的精确** — 间距遵循 4px 栅格，比例遵循明确的层级，对齐精确到子像素。严谨的结构本身传达可靠感。
4. **诚实到字节** — 每一个交互映射到真实的文件操作。`保存` 就是写磁盘，`删除` 就是删文件。没有魔法，没有黑箱。
5. **轻量是道德** — 不引入不必要的依赖。渲染管线 < 50KB，主 Bundle < 100KB gzip，桌面安装包 < 10MB。轻量本身就是对用户的尊重。

## Accessibility & Inclusion

- **标准**：WCAG 2.1 AA
- **对比度**：正文 ≥ 4.5:1，大文本 ≥ 3:1，UI 组件 ≥ 3:1
- **键盘**：所有功能可通过键盘完成。Tab 导航顺序符合视觉布局逻辑
- **屏幕阅读器**：所有交互元素有明确的 ARIA 标签
- **动效**：尊重 `prefers-reduced-motion`。动效不承载功能性信息
- **色觉障碍**：信息不单独依赖颜色传达。区块标记使用形状+颜色双重编码
- **中文优先**：UI 默认中文，英文完整覆盖。中文字体栈包含 PingFang SC / Microsoft YaHei
