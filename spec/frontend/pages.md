## 1. 路由表

```typescript
// src/router/index.ts
const routes = [
  {
    path: '/',
    name: 'home',
    component: NotebookHome,
  },
  {
    path: '/settings',
    name: 'settings',
    component: SettingsPage,
  },
  {
    path: '/:pathMatch(.*)*',
    name: 'not-found',
    component: NotFoundPage,
  },
];
```

| 路径               | 路由名      | 页面组件       | 说明                                           |
| ------------------ | ----------- | -------------- | ---------------------------------------------- |
| `/`                | `home`      | `NotebookHome` | 主编辑页，需要先选择/打开笔记本文件夹          |
| `/settings`        | `settings`  | `SettingsPage` | 全局设置页                                     |
| `/:pathMatch(.*)*` | `not-found` | `NotFoundPage` | 404 兜底页，显示"页面不存在"并提供返回首页链接 |

### 1.1 路由守卫

```
全局前置守卫 (beforeEach):
  1. 检查 useNotebookStore.rootPath 是否为 null
  2. 若为 null 且目标路由不是 /settings:
     → NotebookHome 停留在 INIT 状态，不产生硬重定向
       （INIT 状态本身即是引导选择笔记本，无需额外路由跳转）
  3. 若 rootPath 已设置:
     → 放行，页面正常渲染
```

**设计决策**：不使用嵌套路由（如 `/notebook/:id/note/:path`）。原因：

- MarkLuck 除笔记本打开引导外，不存在"离开编辑器"的导航页面
- 编辑器标签页（Tab 切换笔记）由 `useNotebookStore.openTabs` 管理，不需要路由参数
- 保持 URL 简洁：用户分享链接时只关注笔记本而非文件路径（文件路径是用户本地概念）

### 1.2 顶部导航

所有页面共享一个极简顶部导航栏（`TopNavBar`）：

```
┌──────────────────────────────────────────────────┐
│ 🧠 MarkLuck          [当前笔记本名]    [⚙ 设置]  │
└──────────────────────────────────────────────────┘
```

- 左侧：应用 Logo + 名称（点击回到首页）
- 中间：当前笔记本名称（仅 NotebookHome READY 状态显示，点击可切换笔记本）
- 右侧：设置按钮（路由到 `/settings`）

### 2.1 INIT 状态 — 首次使用 / 未选择笔记本

**触发条件**：

- 首次打开 MarkLuck（无历史笔记本记录）
- 用户关闭了当前笔记本
- `useNotebookStore.rootPath === null`

**用户看到**：

```
┌──────────────────────────────────────────────────┐
│ 🧠 MarkLuck                              [⚙ 设置] │
├──────────────────────────────────────────────────┤
│                                                  │
│                                                  │
│               📓 欢迎使用 MarkLuck                │
│                                                  │
│         轻量化、本地优先的 Markdown 笔记工具        │
│         文件夹即笔记本，数据完全由你掌控            │
│                                                  │
│     ┌──────────────────────────────────┐         │
│     │        📂 打开笔记本文件夹         │         │
│     └──────────────────────────────────┘         │
│                                                  │
│     最近使用的笔记本：                             │
│     ┌──────────────────────────────────┐         │
│     │ 📁 我的笔记        (D:\Notes)     │         │
│     │ 📁 项目文档        (D:\Docs)      │         │
│     │ ...                              │         │
│     └──────────────────────────────────┘         │
│                                                  │
│     或拖拽文件夹到此处                             │
│                                                  │
└──────────────────────────────────────────────────┘
```

**可用交互**：
| 交互 | 效果 |
|------|------|
| 点击"打开笔记本文件夹"按钮 | 触发 `IFileSystemService.openNotebook()` → 弹出系统文件夹选择器 |
| 点击最近笔记本列表中的条目 | 直接打开该笔记本（跳过文件夹选择器，但需验证路径存在） |
| 拖拽文件夹到界面 | 获取拖拽的文件夹路径（仅 Tauri Desktop 支持，Web 端不支持） |
| 点击右上角设置 | 路由到 `/settings` |

**状态转换**：
| 目标状态 | 触发条件 |
|---------|---------|
| `LOADING` | 用户成功选择了文件夹（有权限访问） |
| `ERROR` | 文件夹选择器抛出异常（用户取消选择**不算**ERROR，停留在 INIT） |

**边界情况**：

- **用户取消文件夹选择**：`showDirectoryPicker()` 抛出 `AbortError`，静默捕获，停留在 INIT，不弹错误提示
- **最近笔记本路径不存在**：显示该条目标记为"路径不存在"，hover 显示 tooltip 提示完整路径，点击时弹出提示"该文件夹已不存在，请重新选择"，不自动跳转 LOADING
- **浏览器不支持 File System Access API**：检测 `'showDirectoryPicker' in window`，若不支持则显示降级提示："你的浏览器不支持本地文件夹访问，请使用 Chrome/Edge 或下载桌面版"，并提供"使用 OPFS 沙箱存储"按钮（Firefox/Safari 降级路径）
- **用户拒绝权限**：Web 端再次弹出选择器；信任请求将被浏览器记住

### 2.2 LOADING 状态 — 正在扫描/加载笔记本

**触发条件**：

- `useNotebookStore.openNotebook(rootPath)` 调用后
- 切换笔记本时

**用户看到**：

```
┌──────────────────────────────────────────────────┐
│ 🧠 MarkLuck                           [⚙ 设置]   │
├──────────────────────────────────────────────────┤
│                                                  │
│              ⏳ 正在加载笔记本...                  │
│                                                  │
│              [████████░░░░░░░░] 60%              │
│                                                  │
│              正在扫描: tutorials/javascript.md    │
│              已发现 42 个笔记文件                  │
│                                                  │
│         ┌────────────────────────────┐           │
│         │  📊 加载阶段                │           │
│         │  ✅ 扫描 .md 文件           │           │
│         │  ✅ 读取索引文件            │           │
│         │  🔄 构建文件树              │           │
│         │  ⏳ 加载搜索索引            │           │
│         └────────────────────────────┘           │
│                                                  │
└──────────────────────────────────────────────────┘
```

**可用交互**：
| 交互 | 效果 |
|------|------|
| 点击右上角设置 | 路由到 `/settings`（不中断加载） |
| 取消按钮 | 中止加载 → 回到 INIT 状态 |

**加载阶段**（按顺序执行）：

1. 验证文件夹存在 + 可读 → 设置 `rootPath`
2. 扫描 `.md` 文件（递归）→ 构建 `fileTree`
3. 读取 `.markluck_index.json`（如存在）→ 填充 `useIndexStore`
4. 若索引不存在或有版本不兼容 → 触发索引重建（更新进度条文本）
5. 启动文件监控 `IFileSystemService.watch()`
6. 完成 → 进入 READY 或 EMPTY

**状态转换**：
| 目标状态 | 触发条件 |
|---------|---------|
| `READY` | 扫描完成，至少发现 1 个 `.md` 文件 |
| `EMPTY` | 扫描完成，0 个 `.md` 文件（文件夹存在但空） |
| `ERROR` | 扫描过程中异常（权限不足/路径不存在/IO 错误） |

**边界情况**：

- **超大文件夹（>10,000 个 .md 文件）**：分批加载，每批 500 个文件，虚拟滚动展示。进度条持续更新。用户可以看到部分文件树并开始操作，未加载的部分显示灰色骨架。
- **`.markluck_index.json` 不存在**：新建索引，加载阶段的"读取索引文件"步骤替换为"构建索引"。进度文本更新。
- **`.markluck_index.json` 版本不兼容**：弹出确认提示"索引版本不兼容，需要重建索引（不会影响你的笔记文件）"，用户确认后重建。
- **加载中切换笔记本**：允许，但须等待当前加载完成后执行切换（防并发竞态）。或者用户取消当前加载再选新的。
- **加载超时**：超过 30 秒未完成 → 提示"加载时间较长，是否继续等待？"并提供"取消"和"继续等待"按钮。

### 2.3 READY 状态 — 笔记本已加载，显示编辑器

**触发条件**：笔记本加载完成且存在笔记文件。

这是 MarkLuck 最核心、最复杂的状态，包含多个子状态。

#### 2.3.1 桌面端布局

```
┌──────────┬───────────────────────┬──────────┐
│  文件树   │  标签栏 + 格式工具栏   │  导航树   │
│ (可折叠) │                       │ (可折叠)  │
│          │                       │          │
│ 📁 笔记   │                       │ 📑 标题1  │
│  ├─ 📁子 │   编辑器区域            │  ├─ 标题2 │
│  │  ├─ .md│  (块级混合编辑)        │  └─ 标题3 │
│  │  └─ .md│                       │          │
│  └─ .md  │                       │          │
│          │                       │  🔗 反链  │
│ 🏷 标签云 │                       │  panel   │
│          │                       │          │
│ 🕐 最近  ├───────────────────────┤          │
│   编辑   │  状态栏: 字数 | 行数    │          │
│          │  光标位置 | 保存状态    │          │
└──────────┴───────────────────────┴──────────┘
```

**三栏布局**：

- **左侧栏**（260px~320px）：`SidebarLeft`，包含 `NotebookSelector` + `FileTree` + `RecentNotes` + `TagCloud`
- **中央区域**（flex:1）：`EditorArea`，包含 `TabBar` + `FormatToolbar` + `MarkdownEditor` + `StatusBar`
- **右侧栏**（260px~320px）：`SidebarRight`，包含 `NavTree` + `BacklinksPanel`
- 左右侧栏均可折叠（点击折叠按钮或快捷键 `Ctrl+Shift+L` / `Ctrl+Shift+R`）

**移动端布局**：

- 顶部：`NotebookSelector` + 搜索按钮
- 中央：格式工具栏（横向滚动）+ 编辑器区域
- 底部：状态栏
- 文件树：左侧抽屉式滑出（汉堡菜单触发）
- 导航树 / 反向链接：右侧抽屉式滑出

#### 2.3.2 Sub-State: NO_FILE_SELECTED

**触发条件**：笔记本已加载但用户未打开任何笔记标签页。

**用户看到**：

- 左侧栏：文件树完整展示，标签云和最近编辑列表正常
- 中央区域：空白引导页
  ```
  ┌────────────────────────────────────┐
  │                                    │
  │          📝 选择一篇笔记开始编辑    │
  │                                    │
  │    在左侧文件树中点击任意 .md 文件   │
  │                                    │
  │    或使用快捷键搜索: Ctrl+Shift+F   │
  │    或从最近编辑列表中打开           │
  │                                    │
  │    ┌──────────────────────┐        │
  │    │  ✨ 新建第一篇笔记     │        │
  │    └──────────────────────┘        │
  │                                    │
  └────────────────────────────────────┘
  ```
- 格式工具栏：灰色禁用态（无编辑器实例）
- 右侧栏：隐藏或显示空状态"打开笔记后显示标题导航"

**可用交互**：
| 交互 | 触发方式 | 效果 |
|------|---------|------|
| 打开笔记 | 点击文件树中的 .md 文件 | 创建新 Tab → 进入 FILE_SELECTED |
| 打开笔记 | 点击最近编辑列表条目 | ↑ |
| 打开笔记 | 从搜索结果点击 | ↑ |
| 打开笔记 | 从反向链接点击 | ↑ |
| 新建笔记 | 工具栏 + 按钮 或 右键菜单 | 弹出新建对话框，创建后自动打开 |
| 搜索 | `Ctrl+Shift+F` 或点击搜索按钮 | 打开 SearchDialog → 进入 SEARCHING sub-state |
| 切换笔记本 | 点击 NotebookSelector | 选择新文件夹 → 进入 LOADING |
| 切换笔记本 | 点击设置 → 关闭笔记本 | 进入 INIT |

**状态转换**：

| 目标状态      | 触发条件                                                 |
| ------------- | -------------------------------------------------------- |
| FILE_SELECTED | 用户打开任意 .md 文件                                    |
| SEARCHING     | 用户打开搜索对话框                                       |
| EMPTY         | 用户删除了最后一个笔记文件（外部删除或 MarkLuck 内删除） |
| INIT          | 用户关闭笔记本                                           |
| LOADING       | 用户切换笔记本                                           |

#### 2.3.3 Sub-State: FILE_SELECTED

**触发条件**：至少一个笔记标签页打开，编辑器处于活跃状态。

**用户看到**：

- 中央：活跃的编辑器（CodeMirror 6），包含笔记内容
- 标签栏：显示所有打开的标签页（最多 10 个），活动标签高亮，未保存标签显示圆点
- 格式工具栏：全部按钮可用
- 右侧栏：导航树显示当前笔记标题结构，反向链接面板显示引用当前笔记的列表
- 状态栏：字数统计 / 行数 / 光标行列 / 保存状态（"已保存 14:32" 或 "未保存"）

**可用交互**（核心编辑功能）：

| 功能               | 交互                                | 说明                                    |
| ------------------ | ----------------------------------- | --------------------------------------- |
| **编辑文本**       | 直接输入/删除                       | CodeMirror 捕获所有键盘输入             |
| **格式工具栏**     | 点击工具栏按钮                      | 在光标位置插入对应 Markdown 语法        |
| **快捷键格式**     | `Ctrl+B` 等                         | 与工具栏按钮等价                        |
| **Tab 切换块模式** | 光标位于语法块内按 Tab              | 源码 / 渲染模式切换                     |
| **自动格式识别**   | 输入完整语法块                      | 自动切换到渲染模式                      |
| **保存**           | `Ctrl+S` 或等待自动保存（2秒防抖）  | 写入 .md 文件 + 增量更新索引            |
| **撤销/重做**      | `Ctrl+Z` / `Ctrl+Shift+Z`           | CodeMirror 内置 history                 |
| **切换标签页**     | 点击标签或 `Ctrl+Tab`               | 切换活跃笔记                            |
| **关闭标签页**     | 标签页 × 按钮或 `Ctrl+W`            | 如果未保存则弹窗确认                    |
| **导航树跳转**     | 点击右侧栏标题条目                  | 编辑器滚动到对应位置                    |
| **Wiki-link 跳转** | 点击 `[[笔记名]]`                   | 打开目标笔记（新建 Tab）                |
| **死链创建**       | 点击 `[[不存在的笔记]]`（红色虚线） | 弹出确认"创建新笔记？"                  |
| **标签搜索**       | 点击 `#tag`                         | 触发搜索 `tag:tagname` → 进入 SEARCHING |
| **搜索**           | `Ctrl+Shift+F`                      | 打开全局搜索对话框                      |
| **导出**           | 工具栏导出按钮 或 `Ctrl+Shift+E`    | 打开 ExportDialog → 进入 EXPORTING      |
| **分享**           | 工具栏分享按钮                      | 打开 ShareDialog → 进入 SHARING         |
| **新建笔记**       | 工具栏新建按钮 或右键文件树         | 在当前笔记本中创建新 .md 文件           |
| **删除笔记**       | 右键文件树 → 删除                   | 确认后删除文件，如当前打开则关闭标签页  |

**Tab 标签页管理**：

- 最大标签数：`maxTabs = 10`（可配置）
- 超过上限时：打开新笔记会关闭最久未使用的标签（LRU 淘汰）
- 未保存标签：标签名旁显示实心圆点指示器
- 标签排序：按打开顺序排列，不自动排序
- 关闭逻辑：关闭文件时如果未保存 → 弹出确认对话框

**外部编辑检测**：

- 文件监控发现当前打开的笔记被外部修改 → 编辑器内容自动刷新
- 如果 MarkLuck 有未保存修改 + 外部修改 → 弹出 ConflictDialog：
  ```
  ┌─────────────────────────────────────────┐
  │     ⚠️ 文件冲突                          │
  │                                         │
  │  "tutorials/javascript.md" 已被外部修改   │
  │                                         │
  │  [保留我的版本]  [加载外部版本]  [取消]   │
  └─────────────────────────────────────────┘
  ```

  - "保留我的版本"：保留编辑器当前内容，覆盖外部修改（下次保存时写入）
  - "加载外部版本"：放弃编辑器修改，加载外部文件内容
  - "取消"：关闭对话框，不做任何操作（继续编辑，保存时会再次触发冲突）

**状态转换**：

| 目标状态         | 触发条件                               |
| ---------------- | -------------------------------------- |
| NO_FILE_SELECTED | 用户关闭所有标签页                     |
| SEARCHING        | 用户打开搜索对话框                     |
| EXPORTING        | 用户打开导出对话框                     |
| SHARING          | 用户打开分享对话框                     |
| LOADING          | 用户切换笔记本                         |
| EMPTY            | 用户从外部或 MarkLuck 内删除了所有笔记 |
| INIT             | 用户关闭笔记本                         |

**边界情况**：

- **超大笔记（>5MB）**：编辑器降级为纯文本模式，顶部显示警告条"文件过大，仅显示纯文本"，提供"强制渲染"按钮（点击后使用分块渲染）
- **笔记被外部删除**：如果当前打开的笔记被外部删除，关闭对应的标签页，显示 toast 提示"笔记已被外部删除"。如果有其他打开的标签，自动切换到最近使用的标签
- **网络不可用（PWA）**：所有功能正常。PWA Service Worker 已缓存全量静态资源
- **中文输入法**：IME composition 期间暂停块级解析，避免解析不完整拼音
- **粘贴图片**：`Ctrl+V` 粘贴剪贴板图片 → 自动保存到 `assets/` 文件夹，插入 `![](assets/image-name.png)` 引用

- **清除索引缓存确认**：必须弹出确认框"这将删除所有搜索索引和元数据缓存，笔记文件不会被删除。下次打开笔记本时将重建索引。"
- **切换笔记本后**：设置页不自动关闭，但 NotebookSelector 更新为新笔记本

### 3.3 Saved 状态 — 设置已保存提示

**触发条件**：执行了需要反馈的持久化操作（重建索引、清除缓存等）。

这不作为一个独立的页面状态存在，而是一个 **临时闪现的 UI 元素**。

**用户看到**：

- 在设置页底部滑入一个 toast 通知条（自动消失，3秒）：
  ```
  ┌──────────────────────────────────────────────┐
  │  ✅ 索引重建完成 (127 篇笔记已索引)            │
  └──────────────────────────────────────────────┘
  ```
- 或：
  ```
  ┌──────────────────────────────────────────────┐
  │  ✅ 索引缓存已清除，下次打开笔记本时将重建索引  │
  └──────────────────────────────────────────────┘
  ```

**交互**：toast 可手动关闭（× 按钮），3 秒后自动消失。不影响表单操作。

**状态转换**：toast 消失后回到 Normal。

## 附录 C：状态转换规则（防非预期跳转）

```
所有状态转换必须通过 Pinia Store action 触发，禁止组件直接修改状态字段。

禁止的转换（在路由守卫或 Store action 中拦截）：
  - INIT → FILE_SELECTED (必须经过 LOADING)
  - EMPTY → FILE_SELECTED (必须创建文件后经过 LOADING → READY)
  - ERROR → FILE_SELECTED (必须重新 LOADING → READY)
  - 任何状态 → LOADING (仅在 store.openNotebook() 中触发)

允许的转换：
  - READY 子状态之间可自由切换（SEARCHING / EXPORTING / SHARING 是模态层状态）
  - INIT → LOADING (用户选择文件夹)
  - LOADING → READY / EMPTY / ERROR (加载结果)
  - READY → EMPTY (删除最后一个笔记)
  - EMPTY → LOADING (创建笔记后自动触发)
```
