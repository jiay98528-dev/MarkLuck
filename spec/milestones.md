## 依赖关系

```
M0 ──→ M1 ──→ M2 ──→ M3 ──→ M4 ──→ M5 ──→ M6 ──→ M7 ──→ M10
              │                        │
              └── 索引系统              └── 主题系统（可提前到 M3 并行）
                  被 M3/M4 使用              独立于业务逻辑
```

- M1 → M2：M2 的搜索/标签/Wiki-link 依赖 M1 的渲染管道和文件操作
- M2 → M3：M3 的导出依赖 M2 的索引（导出需要读文件元数据）
- M3 → M4：M4 的模板依赖 M3 的导出基础（复用文件生成逻辑）
- M5（主题）相对独立——可以在 M3 或 M4 之后做，也可以提前并行
- M6（Tauri）依赖 M1-M5 完成——全功能 Web 版就绪后再包装
- M7 是最终收尾
