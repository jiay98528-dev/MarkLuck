## 剩余阻塞

### P0 - WebKit 快速切换笔记仍失败

命令：

```powershell
pnpm --filter @markluck/app exec playwright test e2e/tests/08-edge-cases.spec.ts -g "rapid note switching" --project=webkit --workers=1 --reporter=line
```

结果：失败，测试超时 30000ms。失败点在 `e2e/tests/08-edge-cases.spec.ts:230`：

```ts
await dots.nth(i).click();
```

错误表现：WebKit 等待 `.wing-bookmark-dot`.nth(1) 可见、启用且稳定时持续报告 `element is not stable`，最终超时。全量 WebKit 运行也在同一测试文件附近长时间无进展后被终止。

判定：仍是 RC 阻塞。需要区分是产品动画/布局导致元素稳定性异常，还是测试对“无间隔压力点击”的实现过于依赖 Playwright 稳定性检查。无论哪种，当前三引擎 L3 闸门未闭合。

### P1 - 覆盖率门禁仍未真正达标

`pnpm test:coverage` 已能执行，但总体覆盖率约 23.45% statements/lines，且大量核心 UI/文件/导出模块为 0%。项目规则要求核心模块覆盖率 >= 80%，当前只有“覆盖率命令可跑”，没有形成有效质量闸门。

### P2 - Rust 依赖审计未覆盖

`cargo audit` 仍无法执行，原因是本机未安装 `cargo-audit`：

```text
error: no such command: audit
```

这不是代码失败，但意味着 Rust 依赖安全审计仍未完成。

## GUI 抽样结果

内置浏览器访问 `http://localhost:5173/`，完成以下真实操作：

- 新建空白笔记。
- 输入唯一标题 `GUI-RE-AUDIT-*` 与正文。
- 等待自动保存。
- 刷新页面。
- 在左侧书签中重新打开该笔记，内容可恢复。
- 通过文件抽屉右键菜单删除该笔记。
- 验证左侧书签、文件树、正文均无临时笔记残留，并出现“笔记已删除”提示。

观察到一个非阻塞体验风险：刷新后应用没有自动恢复到刚才编辑的笔记，编辑区回到空白“无标题”；但对应笔记仍存在于书签，点击后内容可正确打开。
