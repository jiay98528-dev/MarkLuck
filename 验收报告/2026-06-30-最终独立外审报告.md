## 结论

本轮阻塞已关闭。上一轮 WebKit `rapid note switching` 的失败不再复现，三引擎全量 E2E 已通过。就本地可执行的发布闸门而言，可以进入 RC 收口。

仍需注意：本机未安装 `cargo-audit`，因此本地 `pnpm audit:rust` 仍无法执行；CI 已配置安装并执行 cargo-audit，最终发布前需要确认 CI 的 Rust audit job 为 green。

## 代码复核

最新修复在 `NotebookHome.vue` 引入 `noteSelectionVersion`，只允许最新选择写入 `activePath/currentContent`。旧的快速点击选择即使完成了异步保存、`statFile` 或 `readFile`，也会在版本检查处退出，避免迟到写回覆盖最后选择。

E2E helper 增加 `focusEditor()`，在输入/追加前显式聚焦 CodeMirror，并等待 `.cm-focused`，用于降低 WebKit 压力状态下键盘输入丢失概率。

## 剩余风险

### P2 - 本机 Rust audit 未验证

命令：

```powershell
pnpm audit:rust
```

结果：

```text
error: no such command: `audit`
```

原因是本机缺少 `cargo-audit`。这不是当前代码失败，但本地外审无法给出 Rust 依赖审计通过结论。CI 已新增安装和执行 job，发布前应以 CI 结果为准。

### P2 - Coverage 门槛是核心服务/工具白名单

当前 coverage threshold 已有效，但覆盖范围仍是核心服务/工具白名单，不是全 UI 核心路径。考虑到三引擎 E2E 已覆盖主要用户旅程，此项不阻塞 RC；后续建议继续提高 `NotebookHome.vue`、编辑器、文件抽屉和导出 UI 的组件/集成测试覆盖。
