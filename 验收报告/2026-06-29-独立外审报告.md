# JotLuck 独立外审报告

> 日期：2026-06-29  
> 基线：`965ff4edb81a8789756840f93929705084604ecd` (`feat(ui): productize UX theme plugin architecture`)  
> 角色：独立测试工程师外审  
> 范围：静态代码、安全配置、Node/Vue 单元测试、Rust/Tauri、三浏览器 E2E、内置浏览器 GUI 用户旅程抽检

## 结论

当前基线不建议作为 RC/发布候选合入发布。核心功能大面积可用，Tauri 安装包可以构建，但质量闸门未闭合：CI 单测 job 必然失败、格式化门禁失败、覆盖率门禁不可运行，三浏览器 E2E 存在稳定失败，且生产代码残留 E2E 全局调试钩子，会把笔记内容暴露到 `window.__jotluck_*`。

## 阻断项

### P0: CI 单元测试 job 必然失败

- 证据：`.github/workflows/ci.yml:69-70` 执行 `pnpm test`。
- 证据：根 `package.json:7-16` 没有 `test` 脚本。
- 实测：`pnpm test` 失败，提示 `Command "test" not found`。
- 影响：CI 的 `test` job 失败，`e2e` job 又依赖 `needs: [lint-and-typecheck, test]`，所以主线 CI 无法通过。

### P0: 格式化门禁失败

- 证据：根 `package.json:13` 使用 `prettier --check "**/*.{ts,vue,css,json,md,yaml}"`。
- 证据：`.prettierignore:1-6` 未排除 `packages/app/src-tauri/target/`、`packages/app/src-tauri/gen/schemas`、`e2e/report/` 等生成物。
- 实测：`pnpm format` 失败，`Code style issues found in 1604 files`。
- 影响：CI `lint-and-typecheck` job 的 Format check 必然失败或高度不稳定。

### P0: 生产代码残留 E2E 全局调试钩子

- 证据：`packages/app/src/components/editor/MarkdownEditor.vue:312` 写入 `window.__jotluck_editorInitValue = props.modelValue`。
- 证据：`packages/app/src/components/editor/MarkdownEditor.vue:320-329` 暴露 `getEditorContent`、`getEditorView`、`predictor`。
- 证据：`packages/app/src/components/editor/MarkdownEditor.vue:398-405` 记录内容前缀到 `window.__jotluck_modelOverwrites`。
- 证据：`packages/app/src/types/tauri.d.ts:14-18` 将这些字段正式声明到全局 `Window`。
- 影响：本地笔记内容、编辑器实例和预测器实例被页面内任意脚本/可信主题代码读取。对“本地优先、不外泄”的产品承诺构成边界风险。

### P0: Chromium 大文档 E2E 稳定失败

- 用例：`e2e/tests/18-blackbox-boundary-performance.spec.ts:303-338`，`B3 large document switches view and remains editable after 200-character input`。
- 首次全量 Chromium：139 passed / 1 failed；失败为视图切换耗时 `1180ms`，超过 `1000ms` 阈值。
- 定向重跑：再次失败，等待 `blackbox-input-marker` 进入编辑器超时。
- 影响：大文档切换视图后可编辑性不可靠，属于真实用户数据输入路径。

### P0: Firefox 快速编辑/切换/刷新 E2E 稳定失败

- 用例：`e2e/tests/18-blackbox-boundary-performance.spec.ts:355-395`，`B4 rapid edit-switch-refresh preserves the edited note without cross-file corruption`。
- 全量 Firefox：136 passed / 3 skipped / 1 failed。
- 定向重跑：同样失败。
- 失败信号：runtime observation gate 捕获 22 条错误/警告，包括 Vue Router 导航错误，以及 Firefox 模块加载失败：`Loading failed for the module ... NotebookHome.vue`。
- 影响：该用例的数据持久化断言通过，但运行时错误不干净；发布前不能忽略。

### P1: WebKit 文字补全 Tab 接受失败

- 用例：`e2e/tests/15-autocomplete-journey.spec.ts:103-132`。
- 定向 WebKit 重跑：失败，按 Tab 后编辑器内容仍为 `这是`，未包含 ghost suggestion。
- 全量 WebKit 在该用例附近出现长时间停滞；后续 cascade failure 来自人为中断服务，不计作产品失败。
- 影响：跨浏览器补全交互不一致。

### P1: 覆盖率门禁不可运行

- 实测：`pnpm --filter @jotluck/app exec vitest run --coverage` 失败。
- 原因：缺少 `@vitest/coverage-v8`。
- 证据：`packages/app/package.json:47-58` devDependencies 中无 coverage provider。

### P1: DOMPurify 存在未修补漏洞

- `pnpm audit --audit-level high` 可通过。
- `pnpm audit --json` 报告 2 low / 1 moderate，均来自 `dompurify@3.4.7`。
- moderate：`GHSA-cmwh-pvxp-8882`，patched `>=3.4.11`。
- 影响：项目的 Markdown/XSS 防线依赖 DOMPurify，建议发布前升级并重跑 XSS/E2E。

## GUI 抽检

### 通过

- 文件抽屉：打开、展开子目录、打开 `笔记A.md`、编辑、等待自动保存，均可观测通过；审计标记已撤销并重新保存。
- 搜索：搜索审计笔记可跳转，跳转后编辑并保存通过。
- Live Preview：点击渲染块进入编辑，输入后 Escape/失焦恢复通过。
- 设置：进入“文字补全”，切换“启用幽灵文本补全”，刷新后状态保持；随后已恢复原值。
- 导出：TXT 导出在 GUI 中显示“已导出文件已保存”。内置浏览器未捕获 download event，视为宿主能力限制；内容正确性由 E2E 覆盖。
- 删除：文件树右键删除审计笔记，确认后左侧书签、文件树、编辑器和搜索索引均无残留。

### 失败/风险

- 新建笔记保存后刷新未恢复当前笔记：内容已写入并可被搜索命中，但刷新后主编辑器回到空白“无标题”状态，左侧书签一度无当前项。对用户表现接近“保存后刷新丢失当前上下文”。
- 文件抽屉关闭行为不稳定：一次切换按钮和 Escape 未立即关闭抽屉，导致左侧最近笔记点击被遮挡；后续状态恢复。建议加入 GUI/E2E 对抽屉打开、关闭、遮罩解除的显式断言。

## 工作树副作用

本次 E2E 会修改已跟踪文件 `e2e/report/index.html`。虽然 `.gitignore:42-44` 忽略了 `e2e/report/`，该文件已经被 Git 跟踪，因此每次跑测试都会污染工作树。当前审计后 `git status --short` 显示：

```text

```

## 建议修复顺序

1. 修复 CI：补根 `test` 脚本或改 workflow 调用 app vitest；修正 `pnpm format` 范围和 `.prettierignore`。
2. 移除或 DEV/E2E 条件保护所有 `window.__jotluck_*` 调试钩子，特别是完整内容和内容前缀。
3. 升级 DOMPurify 至 `>=3.4.11`，重跑 XSS 和全量 E2E。
4. 修复 Chromium B3 大文档切换后不可编辑/性能预算问题。
5. 修复 Firefox B4 的运行时导航/模块加载错误。
6. 修复 WebKit ghost text Tab 接受行为。
7. 补 `@vitest/coverage-v8` 并恢复覆盖率门禁。
8. 将 `e2e/report/index.html` 从 Git 跟踪中移除，保留忽略规则。
