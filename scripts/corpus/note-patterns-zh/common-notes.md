## 每日复盘

今天的结论是：补全质量不是模型越大越好，而是语料越贴近场景越好。MarkLuck 是本地优先的 Markdown 笔记工具，默认补全应该服务记录、整理、复盘和检索，不应该像写作助手那样主动展开剧情或生成大量修辞。

做得好的地方：

- 先删除不合适的来源，再讨论新增内容。
- 把训练入口标准化，避免依赖临时命令。
- 报告中区分模型命中和规则 fallback。

需要改进的地方：

- 语料文件数量还可以继续增加。
- 英文技术笔记需要保持低权重，避免中文上下文里冒出英文片段。
- Markdown 表格语料要控制比例，否则 top context 容易被竖线和空格占满。

明天继续看普通中文补全的噪声。重点检查标点后是否安静、空行是否安静、行中是否只允许结构化补全。
