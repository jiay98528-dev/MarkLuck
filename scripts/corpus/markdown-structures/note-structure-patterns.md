# 笔记结构模式

## 任务清单

- [ ] 确认目标。
- [ ] 阅读相关文档。
- [ ] 列出影响范围。
- [ ] 修改配置。
- [ ] 运行训练。
- [ ] 检查报告。
- [ ] 验证编辑器体感。

补充说明：任务清单适合短句，不适合长段落。每一项最好能直接执行，避免写成含糊的愿望。

## 决策记录

### 背景

当前 baseline 语料分布不合理，长篇文本占比过高。补全系统的目标是帮助用户记录笔记，而不是续写故事。

### 选项

| 选项           | 优点           | 风险           |
| -------------- | -------------- | -------------- |
| 删除小说语料   | 语感更贴近笔记 | 冷启动覆盖下降 |
| 继续降权       | 改动小         | 仍可能污染模型 |
| 增加更多短笔记 | 符合定位       | 需要维护语料   |

### 决定

删除小说语料，补充短笔记和技术笔记。公开网站只作为参考，不复制正文。

### 后续

重新训练 baseline，查看报告分布，再做编辑器体感验证。

## 会议纪要模板

时间：

参与人：

主题：

结论：

- 第一条结论。
- 第二条结论。
- 第三条结论。

风险：

- 当前最大的风险是。
- 需要继续观察的是。

行动项：

- 负责人继续确认。
- 下次会议前给出结果。
- 如果阻塞，及时同步原因。

## 排障模板

### 现象

用户看到的问题是什么，是否稳定复现。

### 复现

1. 打开应用。
2. 进入目标页面。
3. 输入测试内容。
4. 观察结果。

### 预期

说明正常情况下应该发生什么。

### 实际

说明当前发生了什么，最好包含可观察证据。

### 根因

写清楚具体代码位置或数据流断点。

### 修复

说明改了什么，不写无关重构。

### 验证

- 单测通过。
- E2E 通过。
- 手动验证通过。

## 表格记录

| 项目     | 状态   | 备注               |
| -------- | ------ | ------------------ |
| 语料配置 | 已调整 | 小说源已移除       |
| 训练脚本 | 已更新 | 报告增加加权字段   |
| baseline | 待生成 | 需要运行命令       |
| GUI 验收 | 待执行 | 重点看普通中文补全 |

表格适合比较，不适合承载太长的正文。如果备注超过一句话，应该放到表格下面单独说明。

## 引用和提醒

> 注意：公开可读不等于可以复制发布。

> 注意：用户私有笔记只能用于本地增量训练。

> 注意：补全系统越安静，用户越容易信任它。

这些提醒适合放在复盘或设计说明里。它们不需要很长，但要表达清楚边界。
