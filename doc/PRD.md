# MarkLuck 产品需求规格 (PRD)

> 版本：v1.0 | 日期：2026-06-03 | 状态：已确认

---

## 一、产品定义

### 1.1 一句话描述

**MarkLuck** — 轻量化、本地优先、全离线的 Markdown 笔记工具。每一条笔记就是一个 `.md` 文件，文件夹即笔记本。数据完全由用户掌控。

### 1.2 核心价值主张

| 用户痛点                    | MarkLuck 解法                         |
| --------------------------- | ------------------------------------- |
| 笔记数据被云端/专有格式锁死 | 纯 `.md` 文件，任何文本编辑器都能打开 |
| 笔记工具越来越臃肿          | 只做笔记核心功能，非核心不做          |
| 离线不可用                  | 纯本地运行，不依赖任何云服务          |
| 数据迁移困难                | 文件夹即笔记本，迁移 = 复制文件夹     |
| 跨平台体验割裂              | Web + 桌面 + 移动，一套代码全端部署   |

### 1.3 明确不做 (Non-Goals)

- ❌ 思维导图
- ❌ 数据库存储（绝不引入）
- ❌ 云端强制同步
- ❌ 旧版 `.doc` 导出
- ❌ 图形化知识图谱（V1 不做，反向链接文本面板已足够）
- ❌ 协作编辑（多人实时协同）
- ❌ 插件系统（保持内置功能完整）

## 三、功能规格

### F-01: 文件系统与笔记本管理

**描述**：用户指定一个本地文件夹作为"笔记本根目录"。MarkLuck 将该文件夹下的所有 `.md` 文件识别为笔记，子文件夹作为子笔记本。

**行为**：

- 首次启动引导用户选择文件夹
- 文件夹选择后，扫描 `.md` 文件并展示为文件树
- 支持新建笔记/文件夹、重命名、删除、移动
- `.markluck_index.json` 自动维护索引（元数据、标签、反向链接）
- 文件系统变更（外部修改）通过文件监控自动检测并刷新

**边界情况**：

- 空文件夹 → 显示"创建第一篇笔记"引导
- 超大型文件夹（>10,000 个 .md 文件）→ 分批加载 + 虚拟滚动
- 二进制文件（非 .md）→ 不显示在文件树中

**附件管理**：

- 笔记通过相对路径引用 `assets/` 文件夹中的附件
- 支持粘贴图片自动保存到 `assets/`
- 附件与笔记一起随文件夹同步

---

### F-02: Markdown 渲染引擎

**描述**：基于 `marked` + `highlight.js` + `DOMPurify` 的渲染管线。

**支持的语法**：

- 标准 Markdown：标题、段落、加粗、斜体、删除线、链接、图片
- 扩展语法：表格、任务列表、代码块（语法高亮）、LaTeX 数学公式（KaTeX 或 MathJax 轻量版）、脚注
- MarkLuck 特有：Wiki-link `[[笔记名]]`、行内标签 `#tag`

**安全要求**：

- 所有渲染输出必须经过 DOMPurify 清洗
- XSS Payload 自动测试套件覆盖已知绕过向量

**渲染性能**：

- 普通笔记（<100KB）：即时渲染（<50ms）
- 大型笔记（100KB—5MB）：分块渲染 + 虚拟滚动
- 超过 5MB：提示文件过大，仅显示纯文本

---

### F-03: 块级混合编辑器

**描述**：基于 CodeMirror 6 的块级 Markdown 编辑器。用户在单一编辑区域内同时看到渲染结果和 Markdown 源码，按语法块粒度切换。

**核心设计**：

- 后台解析器持续扫描文本，识别"闭环语法块"（如 `**bold**`、`***bold italic***`、`` `code` ``、`[text](url)` 等）
- 每个语法块渲染时，在开头和结尾显示蓝色标记点，标识该块的边界
- **Tab 键切换**：光标聚焦到某个块时，按 Tab 在"源码模式"和"渲染模式"间切换
  - 源码模式：显示原始 Markdown 语法，标记点为蓝色
  - 渲染模式：显示渲染后的富文本，标记点变为绿色
- **自动识别**：用户输入完整语法块的瞬间（如输入 `**bold**` 的第二个 `*`），该块自动切换到渲染模式
- **格式恢复**：切换到源码后如果修改导致语法失效，块下方出现"还原格式"按钮（Tab 快捷键触发），恢复为上一个有效状态
- **新格式适配**：修改源码后如果形成新的有效语法，自动匹配新格式并切换渲染

**格式工具栏**：

- 按钮：加粗、斜体、删除线、标题(H1-H6)、无序列表、有序列表、任务列表、引用、代码块、链接、图片、分割线
- 每个按钮对应 Markdown 语法，点击后在光标位置插入对应语法标记
- 每个按钮显示对应的键盘快捷键提示
- 快捷键参考：`Ctrl+B` 加粗、`Ctrl+I` 斜体、`Ctrl+Shift+S` 删除线、`Ctrl+K` 链接 等

**编辑模式共存**：

- 用户可以通过工具栏/快捷键插入格式（可视化方式）
- 用户也可以直接输入 Markdown 语法（原生方式）
- 两种方式互不冲突，自动识别统一处理

**技术约束**：

- 底层：CodeMirror 6 + 自定义 Decoration/Widget 扩展
- 节流：语法解析 150ms 防抖，避免每次按键都全量重解析
- IME：必须正确处理中文输入法的 composition 事件

---

### F-04: 导航树（左侧栏）

**描述**：桌面端固定在左侧的树状导航面板，展示当前文档的章节标题层级。

**行为**：

- 解析当前 Markdown 文档的标题结构（H1—H6），渲染为可折叠的树状列表
- 点击标题 → 编辑器滚动到对应位置
- 当前编辑位置的标题高亮
- 桌面端：固定在左侧，可折叠
- 移动端/窄屏：抽屉式滑出

**交互**：

- 支持折叠/展开各级标题
- 标题文字过长时省略号截断
- 滚动时自动跟随当前编辑位置

---

### F-05: 搜索系统

**描述**：高级全文搜索，支持中文分词和多种过滤条件。

**搜索能力**：
| 功能 | 描述 |
|------|------|
| 全文搜索 | 所有 `.md` 文件内容检索 |
| 中文分词 | 支持中文词语级搜索 |
| 正则表达式 | `/pattern/flags` 语法触发 |
| 标签过滤 | `tag:javascript` 按标签筛选 |
| 日期范围 | `date:2026-01..2026-06` |
| 文件夹范围 | `folder:tutorials` |
| 组合查询 | 以上条件可任意组合 |

**搜索结果展示**：

- 匹配文本高亮
- 显示上下文片段（匹配前后各 30 字符）
- 按相关度排序（标题匹配 > 正文匹配 > 文件名匹配）
- 点击结果跳转到对应位置

**索引**：

- `.markluck_index.json` 维护全文本索引
- 文件变更时增量更新
- 提供"重建索引"按钮用于修复不一致

---

### F-06: Wiki-link 与反向链接

**描述**：使用 `[[笔记名]]` 语法创建笔记间链接，自动构建反向链接网络。

**行为**：

- `[[笔记名]]` 渲染为可点击链接 → 点击跳转到目标笔记
- 目标笔记不存在时，链接用不同样式标注（死链提示），点击创建新笔记
- 反向链接面板：展示所有引用了当前笔记的其他笔记列表
- 支持 `[[笔记名|显示文字]]` 别名语法
- 支持 `[[笔记名#标题]]` 锚点跳转

**数据流**：

- 索引系统（`.markluck_index.json`）维护全局 Wiki-link 图
- 每次保存时增量更新当前笔记的出链
- 反向链接在打开笔记时实时计算

---

### F-07: 标签系统

**描述**：双重标签语法——YAML frontmatter 结构标签 + `#tag` 行内标签。

**YAML Frontmatter 标签**：

```yaml
---
tags: [javascript, tutorial, beginner]
created: 2026-06-03
---
```

- 结构化标签，用于分类和过滤
- 自动提取到索引

**行内 `#tag` 标签**：

```markdown
今天学习了 #javascript 的闭包概念
```

- 正文任意位置使用
- 渲染为可点击标签（点击触发搜索 `tag:javascript`）

**标签面板**：

- 标签云/列表展示所有标签
- 按使用频率排序
- 点击标签 = 搜索该标签的所有笔记

---

### F-08: 模板系统

**描述**：预置模板 + 自定义模板，支持占位符自动替换。

**内置模板**：

- 日记模板：`{{date}}` `{{time}}` `{{year}}` `{{month}}` `{{day}}`
- 会议纪要模板
- 周报模板

**模板行为**：

- 使用模板创建新笔记时，占位符自动替换为当前值
- 用户可创建自定义 `.md` 模板文件
- 模板存储在笔记本的 `_templates/` 文件夹中（可跨笔记本共享）

---

### F-09: 多格式导出

**描述**：将笔记导出为多种格式。

| 格式     | 实现方式                    | 备注                         |
| -------- | --------------------------- | ---------------------------- |
| PDF      | `window.print()` + 打印 CSS | 零依赖，排版由打印样式表控制 |
| docx     | `docx.js` 库生成            | 保留 Markdown 格式映射       |
| xlsx/CSV | `sheetjs` / 原生 CSV 拼接   | 仅表格内容导出               |
| TXT      | 去除 Markdown 语法标记      | 纯文本提取                   |
| HTML     | 自包含 HTML（内嵌 CSS）     | 用于分享和邮件               |

**导出选项**：

- 是否包含 frontmatter
- 是否包含 Wiki-link（转为纯文本或超链接）
- 代码块是否包含行号
- 图片处理（base64 内嵌或保持相对路径）

---

### F-10: 分享功能

**描述**：将笔记以选定格式分享到外部平台。

**流程**：

```
[分享] → 选择格式（默认PDF） → 选择渠道 → 生成文件 → 发送
```

**格式选项**：

- PDF（推荐，最大兼容性）— 默认选中
- Markdown 源文件
- 自包含 HTML
- 纯文本

**渠道选项**：

- 系统分享面板（微信/QQ/飞书等，由 OS 提供 App 列表）
- 邮件（`mailto:` + 附件）
- 复制到剪贴板
- 导出到本地

**设计原则**：

- 仅本地操作，不上传任何云服务
- 分享 = 生成本地文件 + 渠道分发
- 不存储分享历史或链接

---

### F-11: 编辑历史（最近笔记）

**描述**：快速访问最近编辑过的笔记。

**行为**：

- 记录最近打开的 N 个笔记（默认 20 个）
- 存储在 `.markluck_index.json` 中
- 按最后编辑时间倒序排列
- 展示笔记标题 + 最后编辑时间
- 固定在侧边栏顶部或独立面板

**数据**：

- 仅存储文件路径 + 时间戳
- 文件被删除后自动从历史中移除
- 笔记重命名后更新历史记录

---

### F-12: 外部编辑监控

**描述**：检测文件系统变更，自动刷新编辑器视图。

**行为**：

- 使用 Rust `notify` crate（Tauri 端）或 File System Access API 的 change 事件（Web 端）监控文件
- 当前打开的笔记被外部修改时 → 自动重新加载并提示用户
- 文件被外部删除时 → 关闭对应标签页，提示用户
- 外部新增 `.md` 文件 → 自动更新文件树

**冲突处理**：

- MarkLuck 中有未保存修改 → 外部文件变更 → 弹窗："文件已被外部修改。保留你的版本 / 加载外部版本 / 合并（如果可能）"

---

### F-13: 暗色模式

**描述**：全局暗色主题。

- 支持自动跟随系统主题
- 手动切换：亮色 / 暗色 / 跟随系统
- 所有组件完整适配暗色
- 代码高亮主题随模式切换

### F-15: 系统默认 .md 文件关联

**描述**：MarkLuck 可注册为操作系统中 `.md` 文件的默认打开程序。首次启动时通过欢迎页引导用户设置，后续可在设置中随时更改。

**行为**：

- 首次启动 → 欢迎页 → "将 MarkLuck 设为默认 Markdown 编辑器？"
  - [是] → 注册系统文件关联
  - [否] → 跳过
  - [以后再说] → 后续在设置页提示
- 设置页 → "文件关联" 区域 → 显示当前状态 + [设为默认] / [取消关联] 按钮
- 双击 `.md` 文件 → 在 MarkLuck 中打开（自动定位到文件所在笔记本）

**平台实现**：
| 平台 | 机制 |
|------|------|
| Windows | Tauri `bundle.fileAssociations` → 写入注册表 `HKEY_CLASSES_ROOT\.md` |
| macOS | `Info.plist` 声明 `CFBundleDocumentTypes` + Launch Services |
| Linux | `.desktop` 文件 `MimeType=text/markdown` |
| Web | 不适用 |
| Android | `AndroidManifest.xml` intent-filter |

**边界情况**：

- 系统已有其他默认 `.md` 编辑器 → 注册时系统会弹出选择框（Windows/macOS 原生行为）
- 卸载 MarkLuck → 文件关联失效，系统恢复为之前的默认程序

---

### F-16: VS Code 渲染插件

**描述**：独立发布到 VS Code Marketplace 的 Markdown 渲染插件，复用 MarkLuck 的渲染管线（marked + DOMPurify + highlight.js + Wiki-link / #tag 扩展）。在 VS Code 内提供 `.md` 文件的源码 ↔ 渲染视图切换。

**分发方式**：

- 独立发布到 VS Code Marketplace
- MarkLuck 设置页提供跳转链接引导安装
- 插件也是 MarkLuck 的独立获客渠道

**功能**：

- 注册为 VS Code 自定义编辑器（`CustomTextEditorProvider`）
- 工具栏按钮：源码视图 ↔ 渲染视图（一键切换）
- 渲染视图使用 `@markluck/renderer` 独立包（与主应用共享渲染逻辑）
- 支持所有 MarkLuck 特有的 Markdown 扩展（Wiki-link 渲染、#tag 渲染）
- 代码块语法高亮
- 点击 Wiki-link → 跳转到对应 `.md` 文件（VS Code 内打开）

**架构影响**：

### F-17: 文字补全系统（统一幽灵文本管道）

**描述**：编辑器在用户输入时通过**单一幽灵文本管道**提供智能文字补全。无论用户输入的是普通文字、Markdown 格式语法、Wiki-link 前缀还是标签/路径——系统只做一件事：在光标后显示一条灰色斜体的最佳预测。用户按 `Tab` 一键接受，或继续输入自然覆盖。**无弹出菜单、无下拉选择框。**

**核心设计原则**：系统自动判断当前上下文，融合三个预测源输出唯一最佳建议：
1. **N-gram 统计表** — 基于用户写作习惯的字符/词级频次预测
2. **结构化知识库** — 索引中的笔记名/标签/文件路径（仅特定语法上下文触发）
3. **语法上下文感知** — 检测是否在代码块/frontmatter/列表/格式标记内

**行为**：
- 用户停止输入 ≥ 150ms → 系统分析光标前 2-4 字符上下文 → 在光标后显示灰色斜体幽灵文本（1-20 字符）
- 用户按 `Tab` → ghost text 变为实际文本，统计表即时更新（强化学习）
- 用户继续输入 → ghost text 自动消失，防抖后生成新预测
- 用户按 `Escape` → 清除当前 ghost text（本次会话降低该上下文权重）

**场景示例**：
| 用户输入 | 幽灵文本预测 | 说明 |
|----------|-------------|------|
| `**粗` | `体**` | N-gram 学习到 `**` 后通常跟文字 + 闭合 `**` |
| `[[` | `快速入门]]` | 结构化知识：最近引用的笔记名 |
| `[[快` | `速入门]]` | 结构化前缀匹配 |
| `#` | `javascript ` | 结构化知识：最常用标签 |
| `[text](` | `./notes/readme.md)` | 结构化知识：文件路径前缀匹配 |
| `- ` | `列表项` | N-gram 学习列表上下文常见续写 |

**边界情况**：
- 代码块/frontmatter 区域不触发预测
- 置信度低于阈值时不显示 ghost text
- 空行/纯空白行不触发预测
- 冷启动无历史数据时不显示预测，后台静默构建统计表

**技术约束**：
- 纯前端 TypeScript 实现，零新 npm 依赖（不使用 `@codemirror/autocomplete`）
- 无需 Rust 后端/SQLite/网络
- Web PWA + Tauri Desktop 均可使用
- 首次启动时从 `public/baseline-ngram.v1.compact.txt` 加载出厂基准语料（~15,600 条预训练 N-gram），解决冷启动

**数据持久化**：
- L1 缓存：当前文档的内存统计（打开时 scanDocument，关闭时释放）
- L2 缓存：全局统计 localStorage 持久化（紧凑文本格式，< 5MB）
- 末位淘汰：L2 > 4.5MB 时按 `freq × recencyDecay × sourceWeight` 评分淘汰低频条目
- 基准条目（flag='b'）权重打折，用户不用则自然沉底淘汰

**基准语料训练**：
- 通过 `scripts/train-baseline.ts` 从语料目录 `scripts/corpus/` 自动训练生成
- 语料为纯 Markdown 文件（无需标注），按 `corpus.config.json` 配置的目录和权重扫描
- 仅 P0 格式闭合规则硬编码（Markdown 语法不变），P1-P5 全部语料驱动（可迭代）
- 迭代方式：往 `corpus/` 添加/替换 Markdown 文件 → 运行训练 → 出新基准。无需改代码

**描述**：首次启动时弹出多步骤欢迎向导，介绍 MarkLuck 的核心价值、基础操作，并引导用户完成初始配置。

### F-19: 版本更新检测

**描述**：通过 GitHub API 检查新版本。三道隐私锁止确保用户知情同意。代码签名证书就位前，自动安装选项锁止。

**三锁机制**：

```
锁 1：欢迎页显式授权（默认关闭）
  └─ 锁 2：行为子选项（自动安装 / 仅提醒）
       └─ 锁 3：逐版本静默（"本版本不再提醒" ☐）
```

**行为**：

| 模式 | 检测到新版后 |
|------|-------------|
| **自动下载并安装**（需签名证书） | 后台下载 → 完成弹窗 → 用户确认 → 静默安装 → 自动重启 |
| **仅提醒新版本可用** | 启动 15s 后右下角弹出通知卡片（15s 倒计时条）→ 点击跳转 GitHub Releases |

**技术实现**：
- 前端 `fetch('https://api.github.com/repos/jiay98528-dev/MarkLuck/releases/latest')`
- 24 小时本地缓存，避免频繁请求
- 网络不可用时静默降级，不弹窗
- 证书就位后 Rust 侧 `reqwest` 下载 + NSIS `/S` 静默安装

**设置页控制**：
- 父开关："自动检查可用更新"（默认关闭）
- 子选项（父开关开启时展开）：
  - "自动下载并安装" — 签名证书就位前 **锁止**（显示原因：先行版暂未获得代码签名证书）
  - "仅提醒新版本可用"

**边界情况**：
- 无签名版：自动安装选项锁止，仅"仅提醒"可用
- 签名证书就位后：锁自动解除
- 用户勾选"本版本不再提醒" → 当前版本静默，下一版本恢复通知

### F-20: Markdown 语法速查表

**描述**：编辑页面右下角的悬浮可折叠卡片，展示常用 Markdown 语法格式。可拖拽、不可被删除、不可拖出窗口边界。

**行为**：
- 默认位置：编辑器右下角，折叠状态
- 点击展开/折叠（Tier 2 非线性动画，250-400ms）
- 任意拖拽（pointer 事件），边界检测 clamp 到 viewport 内
- 窗口 < 800px 宽 → 自动折叠，不跟随边界移动
- 不可被删除或隐藏（除非在设置页关闭）
- 位置和折叠状态跨会话持久化（localStorage）

**包含的语法条目**：

| 类别 | 条目 |
|------|------|
| 标题 | `# H1` `## H2` `### H3` |
| 行内格式 | `**粗体**` `*斜体*` `~~删除线~~` `` `代码` `` |
| 列表 | `- 无序` `1. 有序` `- [ ] 任务` |
| 引用与代码 | `> 引用` ` ``` 代码块 ``` ` |
| 链接与图片 | `[文字](url)` `![图片](url)` |
| Wiki-link | `[[笔记名]]` `[[笔记名\|别名]]` |
| 模板 | `{{date}}` `{{time}}` |

**技术实现**：
- 单文件组件 `MarkdownCheatSheet.vue`
- `position: fixed` + `transform` 拖拽定位
- `ResizeObserver` 监听窗口变化
- localStorage 持久化位置/折叠状态/可见性
